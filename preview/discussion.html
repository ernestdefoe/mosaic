Login button doesn't work on Safari 17 after the latest update

Since yesterday's release (build 2.4.0), clicking the "Sign in" button on Safari just spins forever and never completes. Works perfectly in Chrome and Firefox on the same machine.

Console shows this error every time:

Access to fetch at 'https://api.helphub.com/v1/auth/session'
from origin 'https://app.helphub.com' has been blocked by CORS
policy: Response to preflight request doesn't pass access control
check: No 'Access-Control-Allow-Origin' header is present.

Tested on:

  • Safari 17.2 / macOS Sonoma 14.3 — broken
  • Safari 17.1 / macOS Sonoma 14.2 — broken
  • Safari 16.6 / macOS Ventura — works fine
  • Chrome 121, Firefox 123, Edge 121 — all fine

It looks like the preflight OPTIONS request isn't getting the right headers back from the auth endpoint specifically. Other API calls work — only auth is affected.

Happy to share a HAR file if useful. Anyone else seeing this?

Confirmed on our end — this is a regression introduced in 2.4.0 where Safari 17's stricter preflight handling exposed a missing Access-Control-Allow-Credentials header on the /auth/session endpoint when the request originates from a same-site subdomain.

Fix is live as of 2.4.1 (rolled out ~30 minutes ago). No action needed on your side — a hard refresh in Safari (⌘⇧R) should resolve it immediately.

If you're on a custom domain or self-hosted plan, make sure your reverse proxy isn't stripping Access-Control-Allow-Credentials: true from upstream responses. We added a section about this to the SSO troubleshooting doc.

Postmortem will go up on the status page later today. Thanks Marcus for the clean repro and the HAR file — made the fix a 20-minute job instead of a 4-hour one.

For anyone hitting this on a self-hosted setup behind nginx, the missing header is on the response from the auth endpoint, not the request. Quick patch you can drop in your nginx config until you upgrade:

location /v1/auth/ {
  proxy_pass http://upstream;
  add_header 'Access-Control-Allow-Credentials' 'true' always;
  add_header 'Access-Control-Allow-Origin' $http_origin always;
}

Don't forget the always flag — without it the header is only added on 2xx responses, and the preflight returns 204.
